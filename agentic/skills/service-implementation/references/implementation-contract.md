# Implementation Contract

## Inputs

- Task ID or title
- Requirements linked
- Architecture/ADR links
- Owned files/directories
- Interfaces or contracts touched
- Validation commands
- Docs to update

## During Work

- Preserve unrelated changes.
- Follow local style.
- Keep changes scoped.
- Add tests when behavior changes.
- Update documentation when behavior, setup, or architecture changes.

## Output

- Files changed
- Behavior changed
- Validation run
- Validation not run and why
- Risk left
- Handoff target
